1.0 Introduction

 Back to Top

This generated project contains:

  • .java and .xml files that implement a Web Decision service
  • an Ant build file named build.xml which automates the deployment steps
  • a file named build.properties, which is used to parameterize the build.xml file
  • a logging file named logging.properties to describe specific configuration information for handlers.
  • the RuleApp archive file named RuleApp.jar from the source RuleApp project
  • the Execution Unit (XU) configuration file named ra.xml

The component named LeanWSRunner is implemented as a Web service. This Web service is implemented with the aid of WebSphere CE and JAX-WS 2.1.1 RI. As a consequence, the serialization mechanism assumes that the Web service parameters comply with the JavaBean specification.

At initialization time, the *Listener.java class registers the Web decision service in the Rule Execution Server Console using an IlrDecisionServiceMBeanManager instance. When the execution has completed, the *Listener.java class sends the performance metrics to the Rule Execution Server Console.

Ruleset parameters published as Web service methods support:

  • primitive types
  • dynamic types coming from XML Schemas
  • JavaBean classes

2.7 Upgrade your RuleApp archive

If you change the business logic in any way in your rule project, you have to upgrade your RuleApp archive manually for the modifications to be taken into account in this project. To upgrade the RuleApp archive:

  • Deploy the new RuleApp archive and override the ruleapp.jar file generated in this project.
  • Run the deploy.ruleapp Ant target declared in the generated build.xml file.
  • Refresh the project to view the most recent version of the RuleApp deployed in the res_data directory.
